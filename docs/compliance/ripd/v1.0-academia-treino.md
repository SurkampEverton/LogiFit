## 1. Identificação do tratamento

- **Nome do módulo:** Prescrição de treino + execução com Coach (PWA mobile-first ADR 0074) + registro de carga/RPE/foto-vídeo
- **Sprint que entrega:** [Sprint 11 — Academia prescrição treino](../../sprints/11-geral-prescricoes-e-biblioteca.md)
- **ADR de referência:** [ADR 0074 — Coach mobile-first PWA](../../decisions/0074-modo-coach-mobile-first-pwa.md)
- **Marco regulatório:**
  - **Lei 9.696/1998** (CONFEF/CREF — personal trainer/educador físico é profissão regulamentada)
  - **LGPD art. 11 II "a"** (composição corporal e treino podem ser sensíveis quando vinculados a saúde)
- **Finalidade declarada:** prescrever treino personalizado + acompanhar execução com objetividade (carga, RPE, intensidade) + alimentar progressões adaptativas (futuro Sprint pós-35, SaMD II)
- **Categorias de dado tratado:**
  - [x] Frequência de treino / dieta
  - [x] Composição corporal / antropometria (cruzamento com avaliações)
  - [x] Foto/vídeo de execução (mídia opcional)
  - [ ] Saúde clínica (apenas resumo cross-módulo via passaporte se autorizado)
  - [ ] Biometria

## 2. Base legal

| Categoria | Base legal | Justificativa |
|---|---|---|
| Prescrição + execução | art. 7 V (execução de contrato — plano academia) + art. 11 II "a" se vinculado a tratamento clínico | Personal/professor com CREF ativo (`professional_registrations`) prescreve no contrato |
| Foto/vídeo de execução | art. 7 I (consent específico) | Mídia opcional; member pode optar por execução sem registro visual |
| Antropometria periódica | art. 11 II "a" | Indicador de progresso clínico-esportivo |

## 3. Princípios (skeleton)

- **Finalidade:** acompanhamento profissional do treino; **não** treina IA cross-tenant sem opt-in
- **Necessidade:** apenas dados relevantes para evolução (carga, séries, reps, RPE); foto/vídeo opcionais
- **Não discriminação:** member que não consente em foto/vídeo recebe execução baseada em RPE numérico apenas

## 4. Fluxo de dados (a expandir Sprint 11)

```
Personal/professor (CREF ativo + MFA) → prescreve plano em /app/treinos →
  Member abre Coach PWA → /sessao/[id] → registra carga/RPE/series →
    foto/vídeo opcional (scanUpload regra 38) → INSERT execucoes_treino (RLS) →
      audit_log → agregado para progressão (Sprint 11 + futuro adaptativo SaMD II)
```

## 5. Medidas de segurança

| Camada | Controle | Referência |
|---|---|---|
| Isolamento | RLS multi-tenant + scope unit | regra 1 |
| Upload scanning | foto/vídeo via `scanUpload()` | regra 38 |
| MFA | personal/professor MFA obrigatório se feature flag `requires_mfa` | regra 43 |
| Retenção | 5 anos pós-saída do member; member pode pedir exclusão imediata em `/meu/privacidade` | regra 34 |
| Cross-tenant | resumo via passaporte (média/freq) — nunca prescrição bruta + nunca foto/vídeo | regra 42 |

## 6. Avaliação de risco

| Risco | Probabilidade | Impacto | Mitigação |
|---|---|---|---|
| Vazamento de foto/vídeo (constrangedor) | média | médio | Cifrado + signed URL Cloudflare R2 + acesso restrito ao profissional responsável |
| Personal sem CREF ativo prescreve | baixa | médio | Gate `professional_registrations.situation='ativo'` + verificação manual no cadastro |
| Member não consente mas foto é coletada por engano | baixa | médio | UX explícita opt-in + flag em `member_preferences.allow_workout_media` |

## 7. Direitos do titular

- Member vê histórico em `/app/coach/historico` + `/meu/privacidade`
- Edição de RPE pelo próprio member (registro auto-relatado); profissional não pode forjar
- Exclusão imediata de mídia em `/meu/privacidade`
- SLA **15 dias úteis**
