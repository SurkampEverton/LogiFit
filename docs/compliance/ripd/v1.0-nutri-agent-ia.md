## 1. Identificação do tratamento (rascunho)

- **Nome do módulo:** Nutri-Agent IA — agente especializado que lê histórico alimentar + antropometria + objetivos do paciente e propõe ajustes em plano alimentar (sempre como proposta, não execução automática)
- **Sprint que entrega:** Sprint 34
- **ADR de referência:** ADR 0043 + ADR 0044 (esperados, faixa reservada — ver [roadmap.md](../../roadmap.md)). 0043 = arquitetura agente (especializado vs generalizado); 0044 = política de mudança automática (sempre proposta).
- **Finalidade declarada:** apoiar nutricionista (CRN) na revisão e ajuste de planos alimentares, sem substituir a decisão profissional
- **Categorias de dado tratado:**
  - [x] Saúde (histórico clínico, alergias, restrições)
  - [x] Composição corporal / antropometria
  - [x] Diário alimentar (Sprint 31 — Sprint dependência)
  - [x] IA classe SaMD II — comitê IA obrigatório (regra 28)

## Notas críticas (a expandir)

- **Regra 28 obrigatória:** Comitê IA cadastrado no tenant + ata + classificador output + `ai_audit_log` em toda chamada
- **Política "sempre proposta" (ADR 0044 esperado):** Nutri-Agent NUNCA aplica mudança no plano sem confirmação do CRN — `proposeAction` (ADR 0075 + regra 41) com `<ActionConfirmDialog>` obrigatório
- Tool calling tipado via Server Actions (regra 32 + 41) — proibido SQL arbitrário, proibido fetch arbitrário (regra 37)
- BYOK opcional (Sprint 34 + ADR 0064) para tenants que desejam modelo próprio (Anthropic/OpenAI)
- Cross-tenant: Nutri-Agent **nunca lê dado de outro tenant**; resumo cross-tenant via passaporte (regra 42) entra como contexto somente quando paciente liberou
- Direitos do titular: explicação de decisão automatizada (LGPD art. 20) via `/meu/alertas-ia` — cada proposta gerada pelo agente é auditável pelo paciente
- Threat model dedicado em [`docs/threat-models/`](../../threat-models/) — listar quando criado (prompt injection via diário alimentar, alucinação de prescrição, vazamento de plano via tool)
