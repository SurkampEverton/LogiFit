## 1. Identificação do tratamento

- **Nome do módulo:** Cobrança e emissão fiscal — invoices, contratos, cobranças Asaas, NFS-e/NF-e via Focus NFe, remessa contábil
- **Sprints que entregam:** [Sprint 04](../../sprints/04-geral-financeiro-asaas.md) (Asaas), [Sprint 15](../../sprints/15-geral-erp-financeiro-core.md) (NF-e inbox), [Sprint 36](../../sprints/36-geral-fiscal-focus-nfe.md) (emissão fiscal)
- **ADR de referência:** ADR 0014 (esperado, Sprint 04) + [ADR 0057](../../decisions/0057-manifestacao-destinatario-nfe.md) + [ADR 0059](../../decisions/0059-ciclo-fiscal-emissao-focus-nfe.md) + [ADR 0061](../../decisions/0061-motor-retencoes-e-cobertura-fiscal-faseada.md) + [ADR 0066](../../decisions/0066-plano-comercial-pricing-trial.md)
- **Marco regulatório:**
  - **LGPD art. 7º V** (execução de contrato) — base default para cobrança
  - **LGPD art. 11 II "f"** (proteção da saúde) — quando linha de invoice carrega CID/procedimento clínico (TISS Sprint 22)
  - **Lei 9.532/1997 + RFB IN 2.005/2021** — guarda de documentos fiscais 5 anos
  - **LC 116/2003 + RTC 1.400/2016 ABRASF** — NFS-e municipal
  - **Lei 13.787/2018** — quando linha de invoice referencia procedimento de prontuário (retenção 20a)
- **Finalidade declarada:** processar cobrança recorrente e avulsa, emitir documento fiscal obrigatório, manter trilha contábil auditável e atender obrigações fiscais federais/estaduais/municipais
- **Categorias de dado tratado:**
  - [x] Identificação (nome, CPF, endereço, email, telefone do pagador)
  - [x] Financeiro (cartão tokenizado via Asaas, PIX, conta bancária, histórico de pagamento, score interno de inadimplência)
  - [x] Saúde (**indireto**) — descrição de procedimento clínico em linha de invoice quando faturamento TISS (Sprint 22) ou descrição livre de prontuário liga ao financeiro
  - [ ] Biometria
  - [ ] Genética

## 2. Base legal (LGPD art. 7º + art. 11)

| Categoria | Base legal | Justificativa |
|---|---|---|
| Identificação + financeiro | art. 7º V — execução de contrato | Cobrança é cumprimento direto do contrato SaaS LogiFit + contrato tenant↔paciente |
| Histórico de inadimplência | art. 7º VI — exercício regular de direito (cobrança) | Régua D+0/D+21/D+45/D+135 (ADR 0066) é exercício de direito do tenant |
| Procedimento clínico em invoice | art. 11 II "a" + "f" — tutela e proteção da saúde | Linha de invoice carrega CID/procedimento por exigência tributária (ISS) e contábil (DRE) |
| Documento fiscal (NFS-e/NF-e) | art. 7º II — cumprimento de obrigação legal | Lei 9.532/1997 + LC 116/2003 |

## 3. Princípios aplicados (skeleton)

- **Finalidade:** cobrança + obrigação fiscal **exclusivamente**; **não** treina IA (cota Pro/Business para análise de risco interna não exporta dado)
- **Necessidade:** linha de invoice contém apenas o mínimo exigido por SEFAZ/Prefeitura (CFOP, CST, NCM, ISS, ou no caso TISS: CID, TUSS, guia)
- **Segurança:** RLS multi-tenant + criptografia em campos sensíveis (cartão tokenizado Asaas; CPF mascarado em logs via `sanitizeForAlert`) — regras 1, 39
- _Demais princípios serão expandidos quando Sprint 04 implementar Asaas e Sprint 36 fechar Focus NFe_

## 4. Fluxo de dados (a expandir Sprints 04 + 17 + 36)

```
[Tenant operador] → /app/financeiro → wrapAction(createInvoice)
  → INSERT invoices + invoice_lines (RLS por tenant_id) →
    Asaas API (cobrança recorrente / boleto / PIX / cartão) →
      webhook Asaas → UPDATE payments → enfileira emissão fiscal →
        Focus NFe API (NFS-e / NF-e / NFC-e) → autorização SEFAZ/Prefeitura →
          PDF/XML em R2 cifrado + envio email pagador →
            audit_log (hash chain) + WORM anchor 1h
```

- **Origem:** profissional/recepção emite (UI) ou job recorrente roda contrato vencendo
- **Quem acessa:** roles `tenant_owner`, `financeiro`, `recepcao` (RLS); auditor externo via `contador_externo` read-only ([ADR 0061](../../decisions/0061-motor-retencoes-e-cobertura-fiscal-faseada.md))
- **Cross-border:** Asaas (Brasil), Focus NFe (Brasil) — ambos no Brasil, sem TIA. Email transacional via Resend (US) — DPA assinado, dados mínimos (CPF mascarado em corpo)
- **Retenção:**
  - `invoices`/`payments`/`fiscal_emissions`: **5 anos** (RFB IN 2.005/2021) — partition mensal + cold após 2a (regra 34 + ADR 0072)
  - Linha de invoice referenciando procedimento clínico: **20 anos** (Lei 13.787) — herda retenção do prontuário
  - PDF/XML fiscal em R2: **5 anos** (LC 116 + IN RFB), com integridade SHA-256 + WORM
- **Anonimização:** régua de inadimplência D+135 anonimiza identidade do pagador (mantém valor/data para DRE) — ver [`data-deletion-playbook.md`](../data-deletion-playbook.md)

## 5. Medidas de segurança

| Camada | Controle | Referência |
|---|---|---|
| Tokenização | Cartão **nunca** trafega no LogiFit — Asaas tokeniza no front via SDK | PCI-DSS via Asaas |
| Isolamento | RLS multi-tenant + scope `company_id` em invoices (regra 23) | regras 1, 23 |
| Auditoria | `audit_log` append-only + hash chain + WORM | regra 39 |
| MFA | `tenant_owner`, `financeiro` exigem MFA; ações alto-risco (anular invoice, alterar chave Asaas) exigem `requireRecentMfa(<15min)` | regra 43 |
| Sanitização | CPF/cartão/conta mascarados em logs/alerts via `sanitizeForAlert` | ADR 0071 |
| Webhook signature | Asaas + Focus NFe assinam webhooks (HMAC); validação obrigatória em `wrapApiHandler` | regra 33 + 37 |

## 6. Avaliação de risco (skeleton)

| Risco | Probabilidade | Impacto | Mitigação |
|---|---|---|---|
| Vazamento de histórico financeiro de paciente (com CID em linha) | baixa | alto | RLS + criptografia + audit; threat model `pagamento-asaas.md` cobre vetores |
| Cobrança indevida pós-cancelamento | média | médio | Régua de cancelamento + audit + reembolso automático Asaas |
| Webhook Asaas falsificado | baixa | crítico | Validação HMAC obrigatória + threat model `pagamento-asaas.md` |
| Documento fiscal duplicado | baixa | alto | Idempotência por `external_id` + dedup no Focus NFe |

## 7. Direitos do titular (LGPD art. 18)

Implementação plena em Sprint 26. Pré-Sprint 26: titular contata DPO; SLA **15 dias úteis** (Resolução ANPD 2/2024).

**Importante:** apagamento de dado financeiro **não é automático** — preserva-se obrigação fiscal 5a (RFB) + 20a se vinculado a prontuário (Lei 13.787). Direito do titular é **anonimização sem perda de integridade contábil** (manter valor/data, remover identidade).
