## 2. Base legal

| Categoria | Base legal | Justificativa |
|---|---|---|
| Registro alimentar | art. 11 II "a" + consentimento ao aceitar acompanhamento | Paciente registra voluntariamente; nutricionista consome sob CFN 599 |
| Foto do prato | art. 7 I (consent) + art. 11 II "a" | Mídia opcional; armazenamento cifrado |
| Sintomas digestivos | art. 11 II "a" | Insumo clínico para nutricionista |

## 3. Princípios (skeleton)

- **Necessidade:** paciente decide o que registrar; campos obrigatórios mínimos
- **Não discriminação:** Nutri-Agent IA (Sprint 34, SaMD II) não dá interpretação automatizada sem revisão do nutricionista
- **Segurança:** RLS + criptografia at-rest para fotos + audit de acessos do profissional

## 4. Fluxo de dados (a expandir Sprint 31)

```
Paciente (mobile-first, ADR 0063) → Server Action wrapAction →
  INSERT diary_entries (RLS por tenant + paciente owner) →
    se foto: scanUpload (regra 38) → Cloudflare R2 cifrado →
      Nutri-Agent IA (Sprint 34) consome agregado com supervisão humana
```

## 5. Medidas de segurança

| Camada | Controle | Referência |
|---|---|---|
| Isolamento | RLS multi-tenant + scope paciente owner | regra 1 |
| Upload scanning | `scanUpload()` para fotos | regra 38 |
| MFA | Profissional `nutri` MFA obrigatório | regra 43 |
| Retenção | 5 anos pós-alta (alinhada CFN); ou exclusão por paciente em `/meu/privacidade` | regra 34 |
| Cross-tenant | dado bruto **não cruza** tenant; resumo de aderência via passaporte (ADR 0077) | regra 42 |

## 6. Avaliação de risco

| Risco | Probabilidade | Impacto | Mitigação |
|---|---|---|---|
| Vazamento de fotos íntimas (paciente fotografa contexto) | média | alto | Aviso UX explícito + scanUpload + cifrado + permission por profissional |
| Stigma alimentar / transtorno alimentar revelado | média | alto | Acesso restrito ao nutri responsável + ai_audit_log para qualquer leitura cross-role |
| Nutri-Agent IA gera plano sem revisão | baixa | crítico | Gate `withAiClassGate` + supervisão humana bloqueante |
