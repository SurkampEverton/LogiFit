# RIPD v0.1 — Assistente IA Universal (Copilot clínico + chat ancorado)

> **v0.1-skeleton** — RIPD obrigatório (regra 29 + LGPD art. 38). Substância completa será anexada quando [Sprint 06 — Copilot base](../../sprints/06-geral-copilot-base.md) entrar em "doing"; este stub registra escopo e base legal para que o módulo seja reconhecido como coberto pela auditoria de compliance.

> **Distinção importante:** este RIPD cobre o **Assistente IA universal** ([ADR 0075](../../decisions/0075-assistente-ia-universal-tres-camadas-tool-registry.md)) que toca dado clínico em qualquer módulo (anamnese, prontuário, exame). RIPDs específicos por feature IA classe SaMD II+ (ex: pipeline de exames, nutri-agent) ficam separados.

## 1. Identificação do tratamento

- **Nome do módulo:** Assistente IA universal (3 camadas: help/insight/action) — copilot clínico, chat ancorado, sugestão de exercício, resumo de evolução, pré-preenchimento de anamnese
- **Sprints que entregam:** [Sprint 06 — Copilot base](../../sprints/06-geral-copilot-base.md) (camada 1 help) + Sprint 13 (camada 2 insight) + Sprint 24 (camada 3 action via tool registry)
- **ADR de referência:** [ADR 0064](../../decisions/0064-ia-arquitetura-gemini-default-byok-rag.md) (arquitetura Gemini default + BYOK + RAG) + [ADR 0075](../../decisions/0075-assistente-ia-universal-tres-camadas-tool-registry.md) (3 camadas + tool registry) + [ADR 0053](../../decisions/0053-conformidade-cfm-2454-2026-ia-saude.md) (CFM 2.454/2026)
- **Marco regulatório:**
  - **CFM 2.454/2026** (vigência ago/2026) — IA em medicina exige Comitê IA + supervisão humana documentada + classificação SaMD por feature
  - **LGPD art. 11 II "a"** (tutela da saúde)
  - **LGPD art. 20** — direito à revisão de decisão automatizada
  - **ANVISA RDC 657/2022 + 751/2022** — SaMD classificação I-IV; LogiFit evita classe III por design
- **Finalidade declarada:** apoiar profissional na tomada de decisão clínica via sumarização, busca contextual (RAG ancorado em prontuário do tenant), sugestão conservadora (nunca diagnóstica), preenchimento auxiliar de campos estruturados — sempre com revisão humana obrigatória
- **Categorias de dado tratado:**
  - [x] Saúde (prontuário, evolução, exame, queixa, prescrição) — entrada e saída do LLM
  - [x] Identificação (mínima — apenas member_id; nunca CPF/email vai pro provider)
  - [x] Composição corporal (quando módulo academia/nutri usa Copilot)
  - [ ] Genética (fora MVP)
  - [ ] Biometria

## 2. Base legal (LGPD art. 7º + art. 11)

| Categoria | Base legal | Justificativa |
|---|---|---|
| Saúde clínica processada por IA | art. 11 II "a" — **tutela da saúde** + obrigação legal CFM 2.454/2026 | IA atua como ferramenta de apoio profissional; output **nunca substitui** decisão humana |
| Identificação mínima | art. 7º V — execução de contrato (SaaS) | Necessário para contexto da resposta (qual paciente) |
| Logging de auditoria CFM | art. 7º II — cumprimento obrigação legal (CFM 2.454/2026 exige `ai_audit_log`) | Append-only, hash chain, retenção 1a quente + 5a cold |

## 3. Princípios aplicados (skeleton)

- **Finalidade:** apoio decisional **explícito**; classificador de output bloqueia "tem [doença]", "diagnóstico de", "prescrevo X" (regra 28)
- **Necessidade:** RAG ancorado **somente em dados do próprio tenant** (regra 1 + RLS no embedding store); LLM **não** vê outros tenants nem agregados
- **Transparência:** UI marca claramente "Sugestão de IA — revisar antes de usar"; `ai_audit_log` mostra quem aceitou/editou/rejeitou
- **Não-discriminação:** Comitê IA por tenant audita amostras trimestralmente (CFM 2.454/2026); features classe II+ **bloqueadas** sem comitê (gate em feature flag — regra 28)
- **Segurança:** prompt + output + decisão humana gravados em `ai_audit_log`; chave do provider em vault; sanitização anti-PII no prompt outbound
- _Demais princípios serão expandidos no Sprint 06_

## 4. Fluxo de dados (a expandir Sprint 06 + 24)

```
[Profissional] → UI camada IA (chat / botão sugestão / form copilot) →
  Server Action wrapAction(askCopilot, {context_keys}) →
    resolveModelForTask(task, featureKey, tenantCtx) →   // ADR 0064
      RAG: pgvector cache semântico (tenant-isolated) +
            recupera trechos de prontuário/exame do MESMO tenant via RLS →
        sanitizeForLLM(prompt) → remove CPF/email/telefone →
          provider (Gemini Flash default / BYOK Anthropic/OpenAI) →
            outputClassifier(response) → bloqueia termos proibidos →
              ai_audit_log INSERT (input, output, model, prompt_version) →
                UI exibe + botões "aceitar / editar / rejeitar" →
                  decisão humana gravada em ai_audit_log
```

**Camada 3 (action / write):** LLM **nunca** chama Server Action diretamente. Sempre via `proposeAction(toolKey, args)` → registro em `assistant_action_proposals` → UI `<ActionConfirmDialog>` → handler real exige `proposal_id` confirmado válido (regra 41 + ADR 0075).

- **Origem:** profissional (input livre ou estruturado) + RAG do prontuário do tenant
- **Quem acessa:** apenas profissional ativo no tenant (RLS); audit visível ao DPO LogiFit
- **Cross-border:** **SIM** — Gemini (Vertex AI SP — Brasil) ✅ / Groq Whisper (US) — DPA + sanitização / Anthropic (US — BYOK Pro+) — DPA / OpenAI (US — BYOK) — DPA. Ver [`dpo.md`](../dpo.md) sub-processadores.
- **Retenção:**
  - `ai_audit_log` quente: **1 ano** (CFM 2.454/2026 mínimo); cold storage Parquet zstd até **5a** (regra 34 + ADR 0072)
  - Cache semântico pgvector: 90 dias (recovery sem re-chamar LLM)
  - Conversas de chat (não-clínicas): 90 dias
- **Anonimização:** opcional pós-1a — pesquisa interna LogiFit usa cache semântico anonimizado (CPF zerado, member_id hashed) sob acordo expresso

## 5. Medidas de segurança

| Camada | Controle | Referência |
|---|---|---|
| Sanitização prompt | `sanitizeForLLM()` remove PII antes do envio | regra 33 + ADR 0064 |
| Tenant isolation RAG | embedding store com `tenant_id` em RLS — LLM **nunca** vê outro tenant | regra 1 + ADR 0064 |
| Output classifier | bloqueia termos proibidos (diagnóstico, prescrição, "tem doença") | regra 28 + ADR 0053 |
| Comitê IA gate | feature classe II+ não ativa sem `ai_committee_members` cadastrado + ata | regra 28 + ADR 0053 |
| Audit append-only | `ai_audit_log` com hash chain + retenção CFM 1a + cold 5a | regra 39 + regra 34 |
| Tool registry | Action camada 3 só via `proposeAction` + UI confirma; lint `ai-block-respected` | regra 41 + ADR 0075 |
| Quota mensal | hard-stop ao exceder cota do plano (Solo 200 / Starter 500 / Pro 3k / Business 10k / Enterprise 25k) — força BYOK | ADR 0066 + ADR 0064 |
| Rate limit | Upstash Redis por `(tenant_id, user_id, endpoint)` | regra 36 |

## 6. Avaliação de risco (skeleton)

| Risco | Probabilidade | Impacto | Mitigação |
|---|---|---|---|
| LLM cita diagnóstico ("você tem X") | média (sem classifier) | crítico | outputClassifier bloqueia + Comitê IA audita amostras |
| Vazamento de prontuário no prompt para provider US | baixa | alto | sanitizeForLLM remove PII; DPA com providers; cross-border declarado em DPO |
| Tenant sem Comitê ativa feature classe II+ | baixa | alto (CFM violação) | Gate em feature flag + lint CI |
| Prompt injection induz LLM a buscar dado de outro tenant | baixa | crítico | RLS no embedding store; tools server-side; jamais SQL arbitrário do LLM |
| Quota excedida em horário crítico (consulta) | média | médio | UI mostra "cota próxima do limite" + sugere BYOK; degrada para chat sem RAG |
| Decisão automatizada não-revisada usada em conduta clínica | baixa | crítico | UX exige clique "aceitar/editar"; `ai_audit_log` rastreia |

## 7. Direitos do titular (LGPD art. 18 + art. 20)

- **Direito à revisão de decisão automatizada (art. 20):** paciente pode solicitar via DPO revisão de qualquer sugestão IA aceita por profissional sobre seu prontuário. Resposta com cópia do `ai_audit_log` + nome do profissional revisor + decisão final. SLA 15 dias.
- **Direito de oposição (art. 18 §2º):** tenant pode desabilitar IA para um paciente específico (`member_settings.ai_disabled=true`) — Copilot deixa de sugerir nesse contexto.
- **Acesso ao log de IA:** portal `/meu/alertas-ia` (Sprint 26) lista decisões automatizadas envolvendo dados do paciente.
